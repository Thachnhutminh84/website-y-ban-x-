I. ĐIỂM MẠNH CỦA WEBSITE

1. Hệ thống xác thực & phân quyền đa tầng

  • ✦ Xác thực bcrypt (cost 12) — an toàn, khó brute-force
  • ✦ Phân quyền 3 cấp: Admin / Editor (cán bộ) / Viewer (nhân dân)
  • ✦ Hệ thống phê duyệt tài khoản: đăng ký → chờ admin duyệt → mới đăng nhập được
  • ✦ Session regeneration khi login/logout — chống session fixation
  • ✦ Rate limiting đăng nhập: tối đa 5 lần / 300 giây

2. Bảo mật

  • ✦ CSRF token trên form đăng ký và đăng nhập
  • ✦ Prepared statements (mysqli) ngăn SQL injection ở hầu hết các trang
  • ✦ Security headers: X-Frame-Options, X-XSS-Protection, X-Content-Type-Options, CSP
  • ✦ .htaccess chặn truy cập file nhạy cảm: .env, config.php, create-admin.php, phpinfo.php
  • ✦ Session cookie: HttpOnly + SameSite=Strict + strict_mode
  • ✦ Audit logging: ghi lại login, thay đổi dữ liệu, truy cập file, lỗi hệ thống
  • ✦ Soft delete helper với whitelist bảng —防止 SQL injection qua tên bảng
  • ✦ XSS filter cho nội dung tin tức: loại bỏ <script>, on* events, javascript:

3. Kiến trúc mã nguồn

  • ✦ Phân tách rõ ràng: config, auth, UI components (header/footer), business logic
  • ✦ Hệ thống API RESTful: api-search, api-salary, api-bonus, api-contact, api-video...
  • ✦ Environment variables qua .env + EnvLoader class
  • ✦ Cache helper (file-based) với TTL
  • ✦ Breadcrumb system tự động
  • ✦ Quick actions toolbar cho người dùng

4. Chức năng nghiệp vụ

  • ✦ Quản lý tin tức: CRUD đầy đủ, phân loại, phân trang, tìm kiếm
  • ✦ Quản lý video: album, lượt xem, embed YouTube/local
  • ✦ Quản lý phòng ban & nhân sự: 45+ nhân viên, 3 phòng chuyên môn theo NĐ 150
  • ✦ Hệ thống lương: tính lương theo hệ số, ngày nghỉ, xuất Excel theo tháng
  • ✦ Đánh giá hiệu suất: 5 tiêu chí, đánh giá theo quý
  • ✦ Chatbot hỗ trợ người dân
  • ✦ Danh bạ điện thoại, thư viện form/tài liệu
  • ✦ Liên hệ phản ánh: mã ticket tự động, theo dõi trạng thái
  • ✦ Tìm kiếm toàn cục: tin tức, video, nhân sự, phòng ban, lãnh đạo

5. Giao diện người dùng

  • ✦ Responsive design: hoạt động trên desktop, tablet, mobile
  • ✦ CSS variables nhất quán, gradient & animation hiện đại
  • ✦ Font lớn (17-22px) — dễ đọc cho đối tượng người dân
  • ✦ Hero section với thống kê trực quan
  • ✦ Trang Giới thiệu chi tiết về Nghị định 150 và 3 phòng ban mới

II. ĐIỂM YẾU & LỖI BẢO MẬT

1. Lỗi bảo mật NGHIÊM TRỌNG

🔴 CRITICAL: Thông tin admin hardcoded trong mã nguồn
File: create-admin.php — username: admin, password: Mi@131204 (plaintext)
→ Mặc dù .htaccess chặn, nhưng vẫn nằm trong git repo. Nếu deploy sai cách → lộ ngay.
🔴 CRITICAL: MySQL root không có mật khẩu
File: .env — DB_PASS=
→ Database mở toang nếu server có lỗ hổng khác.
🔴 CRITICAL: Deserialization vulnerability
File: cache-helper.php dùng unserialize() trên nội dung file cache
→ Nếu attacker ghi được file cache → PHP object injection → RCE.

2. Lỗi bảo mật CAO

🟠 HIGH: SQL Injection tại xu-ly-them-thanh-vien.php
Dòng 77: WHERE department_id = $department_id (không prepared statement)
→ Biến $department_id được拼 vào SQL trực tiếp.
🟠 HIGH: Thiếu CSRF token trên nhiều POST endpoint
Các trang: quan-ly-phe-duyet.php, quan-ly-nguoi-dung.php, xoa-tin.php
→ Hành động approve/reject/xóa có thể bị CSRF attack.
🟠 HIGH: Rate limiting chỉ dựa vào Session
→ Attacker xóa cookie hoặc dùng session mới → bypass rate limit hoàn toàn.
🟠 HIGH: Script setup/migration không được bảo vệ
run-hr-system.php, run-create-bonus-table.php, setup-department-staff.php
→ Nếu truy cập được → chạy SQL tùy ý.

3. Lỗi bảo mật TRUNG BÌNH

🟡 MEDIUM: IP spoofing trong audit logger
Dùng HTTP_CLIENT_IP và HTTP_X_FORWARDED_FOR → attacker giả IP dễ dàng.
🟡 MEDIUM: Xóa dữ liệu cứng (hard delete)
xoa-tin.php dùng DELETE FROM thay vì soft delete → mất dữ liệu vĩnh viễn.
🟡 MEDIUM: Một số API endpoint không kiểm tra đăng nhập
api-search.php, api-get-video-info.php → data lộ với người lạ.
🟡 MEDIUM: error_reporting(E_ALL) ở một số file
→ Hiển thị lỗi PHP cho người dùng cuối.

4. Vấn đề kiến trúc & mã nguồn

  • ✗ Quá nhiều file trùng lặp: 3-4 phiên bản API cùng chức năng (api-department-staff, api-department-staff-db, api-department-staff-simple)
  • ✗ File backup nằm ngay trong thư mục gốc (index-backup, department-data-backup...)
  • ✗ Hardcoded arrays trong department-data.php thay vì query database
  • ✗ Danh bạ điện thoại dùng mảng PHP cứng trong code thay vì database
  • ✗ Nhiều inline CSS trong PHP (dang-ky.php 300+ dòng CSS)
  • ✗ Không có unit test hay integration test nào
  • ✗ Không có migration system rõ ràng — dùng file SQL + script PHP chạy tay
  • ✗ Thiếu API documentation
  • ✗ Không có HTTPS enforcement
  • ✗ mkdir với quyền 0777 (quá rộng)

IV. ĐỀ XUẤT CẢI TIẾP

Ưu tiên cao (nên làm ngay)

  1. Xóa create-admin.php hoặc chuyển mật khẩu sang .env
  2. Đặt mật khẩu MySQL cho tài khoản root
  3. Thay unserialize() bằng json_decode() trong cache-helper.php
  4. Thêm prepared statement cho $department_id trong xu-ly-them-thanh-vien.php
  5. Thêm CSRF token cho tất cả POST endpoint (phê duyệt, xóa, chỉnh sửa)
  6. Thêm auth check cho các script setup/migration

Ưu tiên trung bình

  1. Thay rate limiting session-based bằng IP-based hoặc database-backed
  2. Sử dụng soft delete thay vì hard delete cho tất cả bảng
  3. Thêm auth check cho API endpoints công khai
  4. Tắt error_reporting ở môi trường production
  5. Xóa các file backup khỏi thư mục gốc
  6. Triển khai HTTPS

Ưu tiên thấp (cải tiến dài hạn)

  1. Loại bỏ file trùng lặp (3-4 phiên bản API)
  2. Chuyển hardcoded arrays sang database
  3. Thêm unit test & integration test
  4. Xây dựng migration system chính thức
  5. Viết API documentation
  6. Tách inline CSS ra file .css riêng
